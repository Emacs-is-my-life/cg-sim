* Purpose

This file records remaining concrete bugs and robustness issues found
after removing items that are intentional simulator design choices:

- fixed llama.cpp trace expansion assumptions

Only actionable correctness and interface issues are listed below.

* P0: State integrity and crash-prevention fixes

** DONE Fix mutation ordering in =sim/core/job/mutation/claim_mutation.py=
CLOSED: [2026-04-01 Wed 21:30]

Current code adds the job to hardware running slots before confirming
that allocation succeeds:

#+begin_src python
for hw in job.running_on:
    hw.run(job)

region = hw.space.claim(job.tensor_id, job.page_idx_start, job.num_pages)
if region is None:
    sys.abort(args)
#+end_src

Required fix:
- perform =hw.space.claim(...)= first
- if claim fails:
  - abort the simulation
  - do not leave the job in any hardware =job_running= list
- only after successful allocation:
  - set =job.region=
  - add the job to hardware running slots

Impact if unfixed:
- failed claim jobs can leave inconsistent hardware running state
- abort paths become harder to reason about

** DONE Validate full input/output region binding in
CLOSED: [2026-04-01 Wed 21:31]
   =sim/core/job/mutation/compute_mutation.py=

Current begin mutation assumes assertion already guaranteed success and
silently proceeds with whatever regions it finds.

Required fix:
- after collecting candidate input regions, verify:

#+begin_src python
len(job.input_regions) == len(node.input_tensors)
#+end_src

- after collecting candidate output regions, verify:

#+begin_src python
len(job.output_regions) == len(node.output_tensors)
#+end_src

- if either check fails:
  - abort with a clear engine/system error
  - do not continue mutating region access state

Recommended implementation detail:
- collect all bindings first
- validate counts
- only then mutate:
  - hardware running slots
  - node status
  - region access flags/counts

Impact if unfixed:
- unexpected assertion/mutation divergence can leave partially mutated
  job or region state
